terraform-azure-application-gateway

Azure Application Gateway v2 with WAF Module

Azure Networking

Executive Summary

Terraform module for deploying Azure Application Gateway v2 with Web Application Firewall (WAF_v2), autoscaling, multi-site hosting with SNI, end-to-end TLS termination, HTTP-to-HTTPS redirects, URL rewrite rules, custom health probes, and zone-redundant deployment. Includes OWASP 3.2 and Bot Manager managed rule sets. Supports dynamic configuration of backend pools, listeners, routing rules, SSL certificates from Key Vault, and user-assigned managed identities.

Data Flow

1
Client sends HTTPS request to the Application Gateway's public IP.
2
WAF inspects the request against OWASP 3.2 and Bot Manager rules; blocks malicious traffic.
3
HTTP listener matches the request by host name and port (multi-site with SNI).
4
Routing rule directs the request to the appropriate backend pool.
5
Rewrite rules modify headers or URL path before forwarding.
6
Application Gateway forwards to healthy backend targets (verified via health probes).

Production Readiness Checklist

  • Enable WAF in Prevention mode
  • Configure SSL certificates from Key Vault
  • Set up HTTP to HTTPS redirect
  • Configure custom health probes for each backend
  • Enable autoscaling with appropriate min/max
  • Deploy across 3 availability zones
  • Enable HTTP/2 for performance
  • Set up managed identity for Key Vault access
  • Configure trusted root certificates for E2E TLS
  • Review WAF rule exclusions for false positives
